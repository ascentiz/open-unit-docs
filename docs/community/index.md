# Community

This section explains how developers, labs, and partners can contribute improvements and provide feedback while the platform is still evolving.

## Principles

- technical clarity is more useful than broad praise
- reproducible issues are easier to act on than vague requests
- safety-relevant feedback should be routed with higher urgency than feature ideas

## In This Section

- [Contributing](contributing.md)
- [Feedback](feedback.md)

## Scope Of Contributions

Useful contributions may include:

- documentation fixes and clarifications
- example improvements
- issue reports with logs and reproduction steps
- API review comments from partner and lab teams
