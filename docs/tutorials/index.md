# Tutorials

This section is for guided workflows that combine platform concepts with practical steps.

## What Tutorials Should Do

Each tutorial should help a developer complete a real task:

- bring up a new device
- validate command and telemetry paths
- deploy an experimental application
- reproduce a lab workflow with clear assumptions

## Initial Tutorials

- [Bring-Up Workflow](bring-up-workflow.md)
- [Safety-Gated Commanding](safety-gated-commanding.md)

## Writing Style

Future tutorials should stay opinionated and practical:

- include prerequisites
- define success criteria
- show expected outputs
- explain safety assumptions explicitly
